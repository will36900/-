# 游戏监测工具 Stage 2 - Windows 使用版

这是“游戏运行健康监测与崩溃诊断工具”的第二阶段版本，面向 Windows 游戏电脑使用。

## 第一次使用

1. 双击 `安装依赖-Windows.bat`
2. 双击 `创建桌面快捷方式-Windows.bat`
3. 以后直接双击桌面的 `游戏监测工具`

如果不想创建桌面快捷方式，也可以直接双击：

```text
启动游戏监测-Windows.bat
```

## 推荐流程

如果你不知道游戏进程名：

1. 先打开游戏
2. 双击桌面的 `游戏监测工具`
3. 选择 `2. 从正在运行的程序中选择`
4. 从列表里选择游戏
5. 程序会开始监控
6. 游戏退出或崩溃后，查看自动生成的 HTML 报告

如果你知道游戏进程名：

1. 选择 `3. 添加游戏档案`
2. 填写游戏名和进程名，例如 `Cyberpunk2077.exe`
3. 下次选择 `1. 开始监控已保存游戏`

## 当前功能

- Windows 双击启动，不需要手动打开终端
- 菜单式终端交互，适合不懂命令行的用户
- 可以从正在运行的程序中选择游戏
- 可以保存游戏档案
- 实时采集游戏进程 CPU、内存、线程数、句柄数
- 采集系统 CPU、内存、磁盘忙碌度
- 游戏退出后自动生成 JSON 和 HTML 诊断报告
- 支持 PresentMon CSV，采集 FPS、1% Low、0.1% Low、帧时间、卡顿次数
- 支持 NVIDIA NVML 和 LibreHardwareMonitor 的可选 GPU 数据采集
- Windows 上尝试读取应用崩溃、显卡驱动、TDR、DXGI 等事件日志
- 基于证据链输出“可能原因”和建议

## FPS/帧时间采集

FPS 采集需要 PresentMon。

在菜单里选择：

```text
5. 配置 FPS/帧时间采集
```

然后填写 `PresentMon.exe` 的完整路径。

如果暂时不配置 PresentMon，程序仍然可以监控 CPU、内存、磁盘、进程退出和部分 Windows 事件。

## 报告在哪里

报告会生成到：

```text
reports
```

菜单里选择：

```text
7. 打开报告文件夹
```

每次游戏退出后会生成：

- `*.html`：给普通用户看的诊断报告
- `*.json`：给高级用户或后续分析用的结构化数据

## 安全边界

本工具不注入游戏、不读取游戏内存、不绕过反作弊系统。FPS 和硬件数据优先通过 PresentMon、NVML、LibreHardwareMonitor、Windows Event Log 这类外部或系统级接口采集。
